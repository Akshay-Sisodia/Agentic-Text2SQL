Database Configuration Guide

This guide explains how to configure and use different database types with the Agentic Text-to-SQL system.

Supported Database Types

The system supports the following database types:

  1. SQLite - Lightweight file-based database (default)
  2. PostgreSQL - Advanced open-source relational database
  3. MySQL - Popular open-source relational database
  4. Microsoft SQL Server - Enterprise-level relational database by Microsoft
  5. Oracle - Enterprise database management system

Configuration Options

There are two ways to configure database connections:

1. Individual Connection Parameters

You can provide separate connection parameters for your database:

  • Database Type: Select from supported database types
  • Database Name: The name of your database
  • Host: The server hosting your database (default: localhost)
  • Port: The port number (depends on database type)
  • Username: Database authentication username
  • Password: Database authentication password

2. Connection URL

Alternatively, you can provide a complete connection URL:

postgresql://user:password@localhost:5432/database
Note: Connection URLs take precedence over individual connection parameters.

Database-Specific Settings

SQLite

SQLite is the simplest to configure and is the default:

  • Only requires a database name (will be stored as a file)
  • No authentication required
  • No host/port required

PostgreSQL

Configuration parameters:

  • Default port: 5432
  • Requires authentication (username/password)
  • Connection URL format: postgresql://user:password@host:5432/dbname

MySQL

Configuration parameters:

  • Default port: 3306
  • Requires authentication (username/password)
  • Connection URL format: mysql://user:password@host:3306/dbname

Microsoft SQL Server

Configuration parameters:

  • Default port: 1433
  • Requires authentication (username/password)
  • Connection URL format: mssql://user:password@host:1433/dbname

Oracle

Configuration parameters:

  • Default port: 1521
  • Requires authentication (username/password)
  • Database name is usually the Oracle SID or service name
  • Connection URL format: oracle://user:password@host:1521/sid

Required Dependencies

Different database types require different drivers. The following dependencies are needed:

  • PostgreSQL: psycopg2-binary
  • MySQL: pymysql
  • MSSQL: pyodbc (requires ODBC Driver for SQL Server)
  • Oracle: cx_oracle (requires Oracle client libraries)

All these dependencies are included in the application's requirements.

Troubleshooting

Common Issues

Database Common Issues Solutions
PostgreSQL Connection refused Check if PostgreSQL service is running, verify port and network settings
MySQL Access denied Verify username/password, check user privileges
MSSQL Cannot load SQL Server driver Ensure ODBC Driver for SQL Server is installed
Oracle DPI-1047: Cannot locate Oracle Client library Install Oracle Client libraries and set environment variables
